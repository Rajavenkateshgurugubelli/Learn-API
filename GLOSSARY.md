A dictionary for API terms.

## A
-   **API (Application Programming Interface)**: A set of rules that allows computers to talk to each other.
-   **Authentication**: Verifying **who** a user is (e.g., Login).
-   **Authorization**: Verifying **what** a user is allowed to do (e.g., Admin vs User).

## B
-   **Body**: The data sent in an HTTP request or response (usually JSON).
-   **Base URL**: The common part of the API address (e.g., `https://api.example.com`).

## C
-   **Client**: The device or program asking for data (Browser, Mobile App).
-   **CRUD**: Create, Read, Update, Delete (matches POST, GET, PUT, DELETE).
-   **CORS (Cross-Origin Resource Sharing)**: A security feature that prevents a website from calling an API on a different domain.

## E
-   **Endpoint**: A specific URL where an API resource is accessed (e.g., `/users`).

## H
-   **Header**: Meta-data sent with a request (like `Content-Type` or `Authorization`).
-   **HTTP Methods**: Actions to perform (GET, POST, PUT, DELETE, PATCH).

## I
-   **Idempotency**: Making the same request multiple times has the same effect as making it once (e.g., DELETE).

## J
-   **JSON (JavaScript Object Notation)**: The standard format for data exchange. `{"hello": "world"}`.
-   **JWT (JSON Web Token)**: A secure way to transmit user identity.

## P
-   **Payload**: Another name for the Body of the request/response.
-   **Pagination**: Splitting large lists of data into pages.

## R
-   **Rate Limiting**: Restricting the number of requests a user can make.
-   **REST**: A popular architectural style for APIs.

## S
-   **Status Code**: A number indicating request success or failure (200, 404, 500).

## W
-   **Webhook**: A "reverse API" where the server calls you when an event happens.
